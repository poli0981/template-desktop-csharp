# Support

Where to get help with this template (and projects spun off from it).

## Quick triage

| You want to… | Use |
|---|---|
| Ask a usage question / share an idea | **GitHub Discussions** on this repo |
| File a bug / request a feature | **GitHub Issues** — pick the matching template |
| Report a security issue | See [SECURITY.md](SECURITY.md). **Not** the issue tracker. |
| Chat informally | Discord (links in [`docs/ABOUT_THE_AUTHOR.md`](docs/ABOUT_THE_AUTHOR.md)) |
| Reach the maintainer directly | `lopop05905@proton.me` (slow, expect a week) |

## Response expectations

This is an **unpaid indie / hobby** project. Reasonable response times:

- Security issues — within 72 hours (see [SECURITY.md](SECURITY.md))
- Bug reports with a repro — within 1 week
- Feature requests — best-effort, no SLA
- "How do I…" questions in Discussions — best-effort, often community-answered

If something is on fire and you have a contract / paid relationship, mention
that in your message; otherwise assume "best effort".

## Before opening an issue

1. Search existing **Issues** *and* **Discussions** for the same problem.
2. Update to the latest release tag and check whether the bug still reproduces.
3. Have a minimal, copy-paste reproduction ready. Bonus points for a failing test.
4. Include `dotnet --info` output and your Windows build (`winver`).

## Discord rules of thumb

- Public channels are not the right place for private bug reports or anything
  that would belong in a Security Advisory.
- Be patient — replies come during ICT evenings (UTC+7), not 24/7.

## Commercial / paid support

Not offered today. If you need it, open a Discussion and we can talk.
